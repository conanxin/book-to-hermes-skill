# Offline Extractive Summary Rules

Session-tested rules for generating chapter summaries without LLM API calls.

## Sentence Splitting

Language-aware delimiter detection:

| Language | Delimiters | Regex |
|----------|-----------|-------|
| Chinese | `。！？；` | `(?<=[。！？；\.\!\?])\s+` |
| English | `.!?` + space/newline | same regex |

Implementation note: The regex uses positive lookbehind to keep the delimiter with the preceding sentence.

## Summary Extraction

Algorithm in `extract_summary()`:

1. Split text into paragraphs by `\n\n`
2. Take first 3 non-empty paragraphs
3. For each paragraph, split into sentences
4. Truncate sentences to 300 chars max (append `...`)
5. Collect up to 5 sentences total
6. Join with spaces
7. Fallback: `(No summary extractable from source.)`

## Key Point Extraction

Priority order in `extract_key_points()`:

1. **Markdown list items** — lines matching `^\s*[-*]\s+(.+)` with length > 10
2. **Numbered list items** — lines matching `^\s*\d+\.\s+(.+)` with length > 10
3. **Sentences from summary** — first 3-5 sentences with length > 15

## Mode-Specific Usage Notes

Generated by `usage_notes_for_mode()`:

| Mode | Guidance |
|------|----------|
| `technical` | Look for implementation details, code examples, architectural decisions |
| `narrative` | Use for thematic analysis, character development, narrative structure |
| `article` | Use for understanding arguments, evidence, background context |
| `api-docs` | Use for interface definitions, parameter descriptions, examples, limitations |
| `project-docs` | Use for architecture overviews, project status, decisions, next steps |

## Chapter File Template

Generated chapters follow this exact structure (no placeholders):

```markdown
# {chapter_title}

## Source Section
Extracted from chapter/section: *{title}*

## Extractive Summary
{summary}

## Key Points
1. {point}
2. {point}
...

## Important Terms
- **{term}** ({type}): {context}
...

## Usage Notes
{mode-specific guidance}

## Connects To
- Other chapters in this skill for related topics.
- External project tools for hands-on implementation.
```

## Verification

Check for zero placeholders after generation:

```bash
grep -ri "\u003c...\u003e\|TODO\|PLACEHOLDER\|TBD\|fill me\|coming soon" \
  ~/.hermes/skills/books/{slug}/
# Expected: 0 matches
```

## Known Limitations

- Table content may leak into summary when tables lack blank-line separation from paragraphs
- Run-on sentences from concatenated paragraphs lose structural cues
- No semantic understanding — purely positional extraction
